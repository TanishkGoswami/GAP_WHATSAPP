**Callback URL**
```text
https://whatsapp.getaipilot.in/webhook
```
Ye tabhi chalega jab ye domain directly backend server ko hit kar raha ho. Backend me route already hai: `GET /webhook` verify ke liye aur `POST /webhook` events ke liye.

**Verify token**
Backend ke deployed `.env` me jo `WA_VERIFY_TOKEN` hai, wahi exact same value yahan paste karni hai.

```env
WA_VERIFY_TOKEN=...
```

Important: `META_APP_SECRET` verify token nahi hai. Wo Meta webhook signature verify ke liye backend me hi rehna chahiye.

**Attach client certificate**
Off hi rehne do, jab tak tumne mTLS/client certificate support explicitly setup nahi kiya.

**Webhook fields**
At minimum `messages` subscribe karo. Isi se incoming messages aur message status updates aayenge.
